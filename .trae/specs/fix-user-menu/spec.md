# 用户菜单与个人中心修复 Spec

## Why
当前右上角用户头像按钮点击后下拉菜单无法正常工作，按钮点击没反应。需要修复下拉菜单交互，并确保"个人中心"和"退出登录"两个按钮功能正常。

## What Changes
- 修复用户头像按钮的下拉菜单交互逻辑，确保点击按钮能正确弹出菜单
- 下拉菜单只保留两个按钮：**个人中心** 和 **退出登录**
- 点击"退出登录"返回登录页面
- 点击"个人中心"跳转到 `/profile` 个人信息资料页面
- 个人中心页面需包含完整的个人信息展示和编辑功能

## Impact
- Affected code: `src/components/Layout.tsx`（用户菜单交互）
- Affected code: `src/pages/ProfilePage.tsx`（个人中心页面，已存在）
- Affected code: `src/App.tsx`（路由，已配置）

## ADDED Requirements

### Requirement: 用户下拉菜单交互
系统 SHALL 在用户点击右上角头像/笔名按钮后，显示包含"个人中心"和"退出登录"两个选项的下拉菜单。

#### Scenario: 打开菜单
- **WHEN** 用户点击右上角头像按钮
- **THEN** 显示下拉菜单，包含"个人中心"和"退出登录"两个按钮

#### Scenario: 点击个人中心
- **WHEN** 用户点击下拉菜单中的"个人中心"
- **THEN** 关闭菜单并跳转到 `/profile` 页面

#### Scenario: 点击退出登录
- **WHEN** 用户点击下拉菜单中的"退出登录"
- **THEN** 清除登录状态并跳转到 `/login` 登录页面

#### Scenario: 点击菜单外部关闭
- **WHEN** 菜单打开状态下用户点击菜单外部区域
- **THEN** 关闭菜单

### Requirement: 个人中心页面
系统 SHALL 提供个人中心页面，用于展示和编辑用户的个人信息资料。

#### Scenario: 查看个人信息
- **WHEN** 用户访问 `/profile` 页面
- **THEN** 显示用户的头像、用户名、笔名、邮箱、个人简介等信息

#### Scenario: 编辑个人信息
- **WHEN** 用户修改个人信息并点击保存
- **THEN** 更新用户信息并提示保存成功
